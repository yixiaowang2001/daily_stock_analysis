# A 股尾盘选股智能体设计

本文沉淀「通过对话完成尾盘选股研究」的产品与工程设计。它不是自动荐股或自动交易系统，而是把策略版本、候选池、风险闸门、逐票评分、次日复盘和策略迭代串成一个可追溯闭环。

## 双层策略边界

尾盘流程必须区分两个策略层，避免把用户的动态选股条件和 Agent 的评分/预测口径混在一起。

### 第一层：尾盘选股策略（用户 / 同花顺动态选股）

这一层面向用户操作，用于在同花顺等工具中生成候选池。它回答的是「哪些股票进入今天尾盘候选池」。

当前一句话选股规则：

> 主板，涨幅 3%-7%，近 20 日有涨停，量比 > 1，换手率 5%-10%，总市值 50 亿-200 亿，股价运行于当日分时均价线上且强于大盘，个股创当日新高且回踩分时均线不破。

结构化条件：

- 上市板块是主板。
- 涨跌幅 >= 3% 且 <= 7%。
- 近 20 日存在涨停或明显涨停基因。
- 量比 > 1。
- 换手率 >= 5% 且 <= 10%。
- 总市值 >= 50 亿且 <= 200 亿。
- 行情价格运行在当日分时均价线上。
- 个股强于大盘 / 跑赢大盘。
- 个股创当日新高后回踩分时均线不破。

这一层只输出候选池，不输出 `watch` / `candidate` / `priority`，也不承载 T+1 开盘预测。

### 第二层：Agent 评估 / 预测策略

这一层面向模型评估，用于对第一层筛出的候选逐票打分、执行风险闸门、输出动作级别、预测 T+1 开盘区间并复盘。它回答的是「这些候选里哪些只是观察、哪些值得进入次日观察池、为什么」。

后续复盘建议必须说明影响哪一层：

- 如果问题来自候选池太宽或条件表达不稳定，才建议修改第一层同花顺选股策略。
- 如果候选进入合理，但模型高估了开盘溢价、承接或题材延续，则优先修改第二层 Agent 评估/预测策略。

## 目标

- 用户可以用自然语言说明当日市场、候选池、偏好和禁区，系统保存为一次实验。
- Agent 基于真实行情、K 线、新闻和策略版本做逐票评分，不编造数据，不强制推荐。
- 评分结果必须结构化，便于前端展示、回测统计和次日复盘。
- 复盘不是证明某次判断对错，而是持续修正策略版本。

## 非目标

- 不自动下单。
- 不承诺收益。
- 不把 `priority` 解读为买入建议。
- 不在没有候选池和风控条件时让模型凭空推荐股票。

## 对话闭环

1. 用户说清楚今天想做什么，例如「今天机器人板块最强，我只看尾盘承接好的 3 只」。
2. 系统抽取并保存 `param_snapshot`：风控模式、验证周期、硬排除条件、评分权重、当日市场上下文。
3. 用户粘贴候选池，系统创建实验。
4. Agent 执行风险闸门，再对候选做结构化逐票评分。
5. 次日自动或手动补早盘指标。
6. Agent 生成复盘、案例摘要和策略修改建议。
7. 用户确认后保存为案例，并按需创建下一版策略；策略建议必须标注修改第一层还是第二层。

## 风险闸门

风险闸门优先于评分。命中以下条件时，候选应降级为 `reject` 或 `watch`：

- ST、退市整理、停牌、临停或上市前 5 日新股。
- 成交额、换手或盘口深度不足。
- 尾盘直线拉升但缺少承接。
- 板块前排走弱、炸板扩散或亏钱效应扩大。
- 公告、监管、业绩或舆情存在未消化的不确定性。

## 评分输出

评估报告先输出面向用户阅读的中文版本，再在末尾附可解析 JSON。阅读版建议包含 `结论卡`、`关键证据`、`主要风险`、`次日开盘预判`、`明日观察/失效条件`，避免让用户先阅读大段结构化字段。

机器解析结果固定写入 `tail_score_result`，每个候选包含：

- `code`
- `score`
- `action_level`
- `confidence`
- `factor_scores`
- `risk_flags`
- `data_quality_flags`
- `next_day_watch`
- `next_open_forecast`
- `invalidation_conditions`
- `one_line_reason`

其中 `action_level` 只能是 `reject` / `watch` / `candidate` / `priority`。
候选池只有一只时也必须独立过风险闸门与评分阈值；如果质量不好，应输出 `reject` 或 `watch`，而不是因为“只有一只”而推荐。

`next_open_forecast` 用于记录 T+1 开盘方向和大致区间，至少包含：

- `direction`：`gap_up` / `flat` / `gap_down` / `uncertain`。
- `expected_open_pct_range`：相对基准价的预期开盘涨跌幅区间。
- `expected_open_price_range`：预期开盘价格区间；数据不足时可为 `null`。
- `base_price`：用于换算价格区间的基准价；14:40 盘中报告可用截点价，盘后报告可用 T 日收盘价。
- `confidence`：`low` / `medium` / `high`。
- `reasoning`：区间判断的核心依据。
- `key_levels`：次日开盘后需要观察的关键价位。

该字段是预测补充，不改变 `score`、`action_level` 或 `passed_candidates`。若报告生成时已经有 T 日收盘后数据，盘后数据只能用于 `next_open_forecast` 和复盘解释，不能倒灌进 14:40 评分。

## 数据截止

- 评分只能使用 T 日 `14:40` 及之前可观察的数据。
- T 日收盘价、全天最高价、收盘成交额、14:40 之后的分时走势都属于盘后或未来信息，只能在复盘中参考，不能用于当时评分。
- T+1 开盘预测属于独立补充项：报告生成于盘后时可以使用 T 日收盘信息估算开盘区间，但必须明确标注依据，且不得反向影响当时评分。
- 候选事实包会显式标记 `selection_cutoff` 与缺失的 14:40 前分钟证据；本地没有分钟证据时，Agent 只能补充受截止时间约束的数据，不能用全日 K 线兜底评分。

## 评分口径

- 市场环境：15 分
- 板块强度：20 分
- 个股量价：25 分
- 流动性：15 分
- 催化质量：15 分
- 风险闸门：最多扣 30 分

分数用于筛选、分层和复盘统计，不用于自动交易。

## 迭代路线

- P0：保存 `param_snapshot`，升级 Agent 评分 JSON 契约，前端可查看对话快照。
- P1：从评分 JSON 中解析因子分，落库并做表格化展示。
- P2：扩展次日指标：开盘溢价、9:30-10:00 冲高、区间回撤、收盘表现、板块延续。
- P3：在问股对话中直接创建策略版本、实验、发起评分、保存复盘。
- P4：用 20/50 次实验统计命中率、失效原因分布和策略版本差异。
